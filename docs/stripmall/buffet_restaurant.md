# Store #2 – Buffet Restaurant (Ponderosa / Golden Corral)

## Hardware and Software Requirements
- **POS**: Custom restaurant management API (Tables/Wait-times/Billing)
- **Kitchen**: IoT-enabled prep stations, freshness tracking sensors, automated supply ordering
- **Display**: Digital menu boards (Solara) and table management tablets (Echo)

## AI Agent Configuration
- **Vale (Menu)**: Data-driven menu optimization based on consumption patterns and costs.
- **Forge (Prep)**: Kitchen coordination and prep schedules based on predicted traffic.
- **Nova (Supply)**: Auto-order ingredients; freshness tracking via IoT.
- **Solara (Loyalty)**: Personalized offers and dining history tracking.
- **Echo (Tables)**: Real-time table and wait-time management.

## Data Schema (Restaurant Sub-ledger)
```json
{
  "timestamp": "ISO8601",
  "type": "revenue | cost",
  "category": "ingredients | labor | dining | maintenance",
  "profit_center": "restaurant",
  "amount": "float64",
  "description": "string"
}
```

## Customer Experience Flow
1. **Entry**: Customer checks in; Echo agent assigns table or provides wait-time estimate.
2. **Dining**: Digital menu boards update in real-time based on kitchen stock (Solara).
3. **Payment**: Loyalty points added to Mirror profile; 30/70 split processed by FundManager.
4. **Follow-up**: Mirror recommends cross-promotions (e.g., "20% off at the VR arcade").

## Staff Training Guide
- **Kitchen Staff**: Executes prep lists generated by Forge; logs ingredient usage via IoT scanners.
- **Service Staff**: Managed by Echo for table assignments; uses Mirror data to personalize guest interactions.
